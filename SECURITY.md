# 보안 정책

## 취약점 제보

보안 취약점이나 자격 증명 노출을 발견하면 공개 Issue를 만들지 말고 저장소의 **Security > Advisories > Report a vulnerability**를 사용해 주세요.

제보에는 영향을 받는 경로, 재현 단계, 예상 영향과 가능한 완화 방법을 포함해 주세요. 실제 사용자 데이터 접근이나 서비스 중단을 유발하는 검증은 삼가 주세요.

## 비밀정보 관리

- 서비스 계정 JSON 키를 저장소에 커밋하지 않습니다.
- 서버는 Application Default Credentials를 사용합니다.
- 로컬 `.env` 파일은 Git에서 제외하고 예제 파일에는 빈 값만 둡니다.
- 노출된 자격 증명은 Git 기록 정리와 별개로 즉시 폐기·교체합니다.
